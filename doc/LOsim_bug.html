PROBLEM:

It is standard for astronomical images to have right ascension (RA) increasing 
to the left. If one takes such an images (generated outside of MAPS) and uses 
it as a "sky model" in MAPS---i.e., runs it through MAPS_im2uv, visgen, and 
maps2uvfits, then images the result---the RA coordinates in the resulting image 
will be mirror-reversed. Furthermore, the sign of the "CDELT1" keyword in the 
header is incorrect, so the absolute coordinates of the sources in the image 
will be incorrect.

This problem does not occur if the input "sky model" was generated using LOsim, 
since images generated by LOsim have RA increasing to the right. Thus if LOsim 
images are run through MAPS_im2uv, visgen, and maps2uvfits and then imaged, the 
coordinates will be handled correctly.
